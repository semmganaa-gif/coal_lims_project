## 1. DATABASE MODELS

### Equipment Model (`app/models.py:640-759`)

**Хүснэгтийн нэр**: `equipment`

**Үндсэн талбарууд**:
- `id` (PK): Integer
- `name` (String 150, NOT NULL): Тоног төхөөрөмжийн нэр
- `manufacturer`, `model`, `serial_number`: Техникийн мэдээлэл
- `lab_code` (String 50): Дотоод код
- `quantity` (Integer, default=1): Тоо ширхэг
- `location`, `room_number`: Байршил
- `related_analysis` (String 200): Холбогдох шинжилгээ (Ж: "Aad,Mad,CV")
- `status` (String 20, indexed): 'normal', 'broken', 'needs_spare', 'maintenance', 'retired'
- `calibration_date`, `calibration_cycle_days`, `next_calibration_date`: Шалгалт тохируулга
- `category` (String 50, indexed): 8 категори
- `register_type` (String 30, indexed): 7 төрлийн журнал
- `extra_data` (JSON): Нэмэлт талбарууд

**Категориуд** (8):
1. `furnace` - Зуух
2. `prep` - Бэлтгэл
3. `analysis` - Шинжилгээний
4. `balance` - Жин
5. `water` - Усны лаб
6. `micro` - Микробиологи
7. `wtl` - WTL/BSL
8. `other` - Бусад

**Register Types** (7 журнал):
1. `main` - Үндсэн бүртгэл
2. `measurement` - Хэмжих хэрэгсэл
3. `glassware` - Шилэн эдлэл
4. `internal_check` - Дотоод шалгалт
5. `new_equipment` - Шинэ тоног төхөөрөмж
6. `out_of_service` - Ашиглалтаас гарсан
7. `balances` - Жингийн бүртгэл

### MaintenanceLog Model (`app/models.py:761-803`)

**Хүснэгтийн нэр**: `maintenance_logs`

**Талбарууд**:
- `id` (PK): Integer
- `equipment_id` (FK → equipment.id): Тоног төхөөрөмж
- `action_date` (DateTime, indexed): Үйлдлийн огноо
- `action_type` (String 50): 'Calibration', 'Repair', 'Maintenance', 'Daily Check'
- `description` (Text): Тайлбар
- `performed_by` (String 50): Гүйцэтгэсэн
- `certificate_no` (String 50): Гэрчилгээний дугаар
- `result` (String 20): 'Pass', 'Fail', 'Warning'
- `file_path` (String 256): Гэрчилгээний файл
